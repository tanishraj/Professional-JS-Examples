Location Object

One of the most useful object in the BOM is location, which is a property of both window object and document object(this is where a lack of standards leads to some real confusion). The location object represents the URL loaded in a new window and, as an added bonus, it also parses the URL into various segments.

Properties

  • hash: if the URL contains a pound sign(#), this returns the content after it. (for example: http://www.somewhere.com/index#section1) has a hash equal to #secton1.
  • host: The name of the server.
  • hostname: Most often equal to host, this sometimes eliminates the www.from the front.
  • href: The full url of the currently loaded page.
  • pathname: Everything after the host in the URL. for example: the path for http://www.somewhere.com/pictures/index.html is /pictures/index.html
  • port: The port of the request if specified in the URL. By default most URLs don't include the port as part of the URL so this property is typically blank.
  • protocol: The protocol used in the URL. This is everything before the two forward slashes in the URL.
  • search: otherwise known as query string. This is everything after a question mark in a URL performing a GET request.
